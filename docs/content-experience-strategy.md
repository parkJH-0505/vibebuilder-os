## 3개 관점의 교차 합의

### 모두가 동의한 것

1. **ch01(빈 껍데기)이 가장 시급** — 5가지 구성요소 프레임워크가 16챕터 내내 참조되는데, 시각적 각인 장치가 전무
2. **비유가 있는 곳에 비주얼이 필요** — 식당, 호텔 키카드, 빙산, 피자 라인 등 풍부한 메타포가 텍스트로만 소비됨
3. **Mermaid 확충이 가장 빠른 승리** — 이미 컴포넌트가 있고, ch02~04에서 효과 검증됨
4. **4~5 스크롤마다 인지 채널 전환** — 연속 텍스트 최대 길이를 제한해야 함
5. **1인 유지보수 가능성이 제약 조건** — 복잡한 상태 관리나 외부 의존성 최소화

### 관점별 고유 기여

| 전문가 | 핵심 기여 |
|--------|----------|
| **UX 전략가** | 에셋 6유형 분류 체계(구조/흐름/비교/일러스트/스크린샷/요약카드), 읽기 리듬 설계 가이드라인, 콘텐츠 에디팅 5원칙 |
| **프론트엔드** | 10개 신규 컴포넌트 제안 + MDX 문법 예시, 기존 컴포넌트 개선안, 레이아웃 패턴 3종, 기술적 제약 분석 |
| **비주얼 에디터** | 챕터별 52개 구체적 에셋 매핑(27 P1 + 18 P2 + 7 P3), 스타일 가이드, AI 이미지 프롬프트 가이드라인 |

## 읽기 리듬 설계 원칙

### 이상적 챕터 구조

```
[도입 텍스트 — 문제 제기, 공감] (2~3 문단)

[비주얼 #1] — 개념 일러스트 or 핵심 비유 시각화
  → "아, 이런 이야기구나" 직관적 파악

[핵심 설명 텍스트] (3~4 문단)
[Callout — 핵심 포인트 강조]

[비주얼 #2] — 흐름 다이어그램 or 구조 다이어그램
  → 텍스트로 설명한 프로세스를 한눈에 정리

[심화 설명 텍스트] (2~3 문단)
[SelfCheck — 학습자 자기 점검]

[비주얼 #3] — 비교 카드, 인터랙티브, 체크리스트
  → 실전 적용 맥락의 참고 자료

[실전 예시 / 사례] (2~3 문단)

[비주얼 #4] — 요약 카드 or 대표 비주얼
  → 챕터 핵심을 1장으로 (기억 앵커)

[졸업 테스트 + ActionItem + NextPreview]
```

### 핵심 규칙

1. **4~5 스크롤(800~1000px) 안에 인지 채널 전환** — 텍스트 → 다이어그램, 텍스트 → 인터랙션 등
2. **한 챕터, 한 대표 비주얼** — 나중에 "그 챕터에서 뭘 배웠더라?"에 떠오르는 기억 앵커
3. **비주얼은 텍스트를 반복하지 않고 변환한다** — 텍스트가 "무엇"을, 비주얼이 "관계/구조/흐름"을
4. **비유가 있으면 반드시 시각화 검토** — 특히 이후 챕터에서 재참조되는 비유는 필수

## 콘텐츠 첨삭 원칙

텍스트를 그대로 쓰는 것이 아니라, **에셋과 텍스트를 조합해 더 효과적으로 전달**한다.

1. **텍스트를 줄이고 에셋으로 대체할 수 있는 곳을 적극 찾아라** — 3문단으로 설명하는 프로세스를 StepByStep 컴포넌트 1개가 대체 가능
2. **비교 테이블 5행 초과 시 시각적 대안 병행** — 테이블 유지 + 상단에 포지셔닝 맵/차트 추가
3. **번호 목록은 StepByStep이나 Timeline으로 전환 검토** — 특히 순서가 중요한 프로세스
4. **"A vs B" 구조는 TwoColumn이나 BeforeAfterCompare로** — 텍스트 비교보다 시각적 대비가 효과적
5. **코드 블록은 주석과 하이라이트로 가독성 향상** — 비개발자에게 "어디를 봐야 하는지" 시선 유도

4. **4~5 스크롤마다 인지 채널을 전환하라** — 연속 텍스트는 학습자를 이탈시킨다
5. **크로스 챕터 재활용 에셋이 ROI 최고** — 한 번 만들면 여러 챕터의 품질을 동시에 향상
